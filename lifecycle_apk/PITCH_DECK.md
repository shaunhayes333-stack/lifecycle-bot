# SLIDE 6: PRE-PROPOSAL KILL SYSTEM

## Stop Wasting Cycles on Garbage

### The Problem (V3.1)
```
Garbage token → CANDIDATE → PROPOSED → SIZING → FDG KILL
                    ↑                              ↑
                    └── Wasted compute cycles ─────┘
```

### The Solution (V3.2)
```
Garbage token → PRE-PROPOSAL KILL → SHADOW-TRACK
                        ↑
                        └── Immediate rejection, learning preserved
```

### Pre-Proposal Kill Triggers:
- **C-grade + confidence < 35%** → Straight to shadow-track
- **C-grade + memory ≤ -8** → Straight to shadow-track
- **Liquidity < $2,000** → Ineligible for watchlist
- **Liquidity < $10,000** → Ineligible for execution

**Result:** Cleaner logs, faster decisions, preserved learning.
